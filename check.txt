========================================
任务一：多模态情感感知与识别
========================================

[已实现] ASR：语音转文本
- 采用：SenseVoice / FunASR AutoModel
- 相关脚本：
  - project/scripts/sensevoice_asr_ser.py
  - project/scripts/pipeline_e2e_demo.py
  - project/scripts/web_demo_app.py
- 输出字段：
  - asr_text

[已实现] SER：语音情感识别
- 采用：SenseVoice emotion token，例如 <|NEUTRAL|>
- 映射到四类：
  - happy
  - sad
  - angry
  - neutral
- 输出字段：
  - audio_emotion_raw
  - audio_emotion

[已实现] 文本情感识别
- 当前采用本地开源模型：
  - project/models/Chinese-Emotion-Small
- 新增模块：
  - project/scripts/text_emotion_model.py
- 输出字段：
  - text_emotion
  - text_confidence
  - text_confidence_type = softmax_probability
  - text_emotion_raw
  - text_emotion_scores
- 说明：
  - text_confidence 来自模型 softmax 概率，不再使用手写规则伪装置信度。
  - 规则版仅作为 fallback，fallback 时 text_confidence 为 null，另给 text_rule_score。

[已实现] Qwen 语义情绪判断
- 采用：本地 Qwen3-8B GGUF，经 llama-server OpenAI 兼容接口调用
- 目的：
  - 弥补文本情感模型对“担心、焦虑、压力、失败风险”等语义识别不足的问题。
- 输出字段：
  - semantic_emotion
  - semantic_confidence
  - semantic_confidence_type = llm_self_estimated
  - semantic_intent
  - semantic_valence
  - semantic_arousal
  - semantic_evidence
  - semantic_reason
- 注意：
  - semantic_confidence 是 LLM 自评置信度，不等同于 softmax 概率。

[已实现] 跨模态融合仲裁
- 当前融合来源：
  1. SenseVoice audio_emotion
  2. Chinese-Emotion-Small text_emotion + text_confidence
  3. Qwen semantic_emotion + semantic_confidence
- 典型规则：
  - 语音 neutral，但 Qwen 语义为 sad/angry 且置信较高，则采用语义情绪。
  - 文本模型 neutral，但 Qwen 语义识别出担心、压力、焦虑、愤怒，则采用语义情绪。
  - 语音 angry 等高唤醒情绪与文本 neutral 冲突时，倾向保留语音高唤醒信号。
  - 多源一致时直接采用一致情绪。
- 输出字段：
  - final_emotion
  - fusion_reason

[未实现/可选] 视觉模态补充
- 文档中任务四为可选模块，目前尚未接入视觉表情、姿态或口型。

========================================
任务二：情感认知与对话生成
========================================

[已实现] 基于情感的共情回复生成
- 采用：本地 Qwen3-8B GGUF + llama-server OpenAI 兼容接口
- 入口：
  - project/scripts/web_demo_app.py
  - project/scripts/pipeline_e2e_demo.py
- Web Demo 当前回复逻辑：
  - 普通问题直接回答。
  - 用户表达难受、担心、生气时，先回应情绪，再给一句有用支持。
  - 用户明确问怎么办时，再给具体建议。
  - 用户感谢、确认或打招呼时，简短回应。
  - 不编造用户身份。
  - 不把所有话题都套成任务/作业/步骤建议。
- 稳定性：
  - 要求 LLM 输出 JSON。
  - 若 LLM 未返回 JSON，Web Demo 会容错，避免整轮请求失败。

[已实现] 结构化标签输出
- 输出结构：
  {"emotion":"happy|sad|angry|neutral","reply_text":"..."}
- emotion 会强制对齐到 final_emotion，用于驱动 TTS 参数。

[已实现] 上下文对话
- Web Demo 支持多轮对话。
- 每个浏览器会话窗口有独立 session_id。
- 后端根据 session_id 保存和恢复上下文。

[已实现] 会话持久化
- 默认目录：
  - project/outputs/web_demo/sessions
- 文件结构：
  - index.json
  - <session_id>.json
- 每个 session JSON 保存：
  - 会话标题
  - pinned 置顶状态
  - 创建/更新时间
  - 每轮用户输入
  - 每轮 AI 回复
  - 完整响应 JSON
  - TTS 音频 URL
- 前端接口：
  - GET /api/sessions
  - GET /api/sessions/{session_id}
  - POST /api/sessions/{session_id}/rename
  - POST /api/sessions/{session_id}/pin
  - DELETE /api/sessions/{session_id}

- 支持：

  - --use-voice-id
  - --use-voice-cache
  - 固定参考音频 ref_audio/ref_text

[已实现] Web Demo TTS 播放
- Web Demo 可选生成 TTS。
- 生成后可在网页内播放。
- TTS 错误会回传到页面。

[当前策略] 高保真优先
- tts_qwen3_from_pipeline.py 默认禁用后处理。
- Web Demo 调用 TTS 时默认带 --disable-postprocess。
- 原因：
  - 后处理变速/变调会带来电音、回声感风险。
  - 当前优先保证可演示音质。

[待增强] 真正情感化声音控制
- 当前 emotion 主要影响参数记录和潜在 TTS 控制。
- 更稳妥的后续方案：
  - 准备 happy/sad/angry/neutral 四类固定参考音频。
  - 根据 final_emotion 选择对应 ref_audio/ref_text。

========================================
当前端到端 Web Demo 链路
========================================

文本输入或音频输入
-> SenseVoice ASR/SER
-> Chinese-Emotion-Small 文本情感识别
-> Qwen 语义情绪判断
-> 情感计算规则仲裁
-> Qwen 共情回复 JSON
-> TTS 参数生成
-> Qwen3-TTS 生成音频
-> Web 页面展示与播放
-> 会话完整响应写入 session JSON

========================================
常用启动命令
========================================

一行启动 Web Demo：
conda activate sensevoice; python "D:\0digi-human\project\scripts\web_demo_app.py"

显式指定会话目录：
conda activate sensevoice; python "D:\0digi-human\project\scripts\web_demo_app.py" --session-dir "D:\0digi-human\project\outputs\web_demo\sessions"

关闭 Qwen 语义情绪判断：
conda activate sensevoice; python "D:\0digi-human\project\scripts\web_demo_app.py" --disable-llm-semantic-emotion

测试文本情感模型：
conda run -n sensevoice python project\scripts\text_emotion_model.py "我现在很担心任务完成不了" --model "D:\0digi-human\project\models\Chinese-Emotion-Small" --device cuda:0
